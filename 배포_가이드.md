## 🔧 1단계: Node.js 설치 확인

Firebase 배포 명령어는 Node.js 위에서 동작합니다.

### 확인 방법
1. **`Win + R`** → `cmd` 입력 → 엔터 (명령 프롬프트 열기)
2. 아래 명령어 입력:
   ```bash
   node -v
   ```
3. **`v18.x.x`** 같은 버전이 나오면 ✅ 설치되어 있는 거예요. 4단계로 가세요.
4. **`'node'은(는) 인식되지 않습니다`** 가 나오면 ❌ 설치 필요. 2단계로 가세요.

---

## 📥 2단계: Node.js 설치 (이미 있으면 건너뛰기)

1. 브라우저에서 https://nodejs.org 접속
2. **LTS 버전** (왼쪽 초록 버튼) 다운로드
3. 다운로드된 `.msi` 파일 더블클릭 → **계속 Next 누르기** (기본 옵션 그대로)
4. 설치 완료 후 **명령 프롬프트를 닫고 새로 열기** (중요!)
5. 다시 `node -v` 입력해서 버전이 나오는지 확인

## 🔥 3단계: Firebase CLI 설치

명령 프롬프트에서:

```bash
npm install -g firebase-tools
```

⏰ 1~2분 정도 걸립니다. 영어로 노란 글씨가 나와도 빨간 ERROR만 없으면 OK!

설치 확인:
```bash
firebase --version
```
→ `13.x.x` 같은 게 나오면 성공.

> **권한 오류가 나면?** 명령 프롬프트를 **관리자 권한으로** 다시 열기:
> 시작 메뉴 → "명령 프롬프트" 우클릭 → "관리자 권한으로 실행"

## 🔑 5단계: Firebase 로그인

```bash
firebase login
```

- 브라우저가 자동으로 열립니다.
- Firebase 콘솔에 사용 중인 **구글 계정으로 로그인**.
- "Allow" 또는 "허용" 클릭.
- 명령 프롬프트로 돌아오면 `✓ Success!` 메시지가 보입니다.

## 🛠️ 6단계: Firebase 프로젝트 초기화

같은 폴더에서:

```bash
firebase init hosting
```

질문이 차례로 나옵니다. 아래대로 답하세요:

| 질문 | 답 |
|---|---|
| Are you ready to proceed? | **Y** + 엔터 |
| Please select an option | **Use an existing project** 선택 (방향키 ↓ + 엔터) |
| Select a default Firebase project | **bing-go-game** 선택 + 엔터 |
| What do you want to use as your public directory? | **`.`** (점 하나) 입력 + 엔터 |
| Configure as a single-page app? | **N** + 엔터 |
| Set up automatic builds with GitHub? | **N** + 엔터 |
| File ./index.html already exists. Overwrite? | **N** + 엔터 (덮어쓰지 않기) |

> 💡 `public directory`를 `.` 으로 한 이유: 현재 폴더 자체를 그대로 호스팅에 올리려고요.

완료되면 폴더에 `firebase.json`, `.firebaserc` 파일이 새로 생깁니다.

---

## ✏️ 7단계: 배포 설정 미세 조정

폴더 안의 `firebase.json` 파일을 메모장으로 열고, 아래처럼 수정:

```json
{
  "hosting": {
    "public": ".",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**",
      "배포_가이드.md"
    ],
    "rewrites": [
      { "source": "**", "destination": "/bingo_game.html" }
    ]
  }
}
```

저장 후 닫기.

> 이렇게 하면 누가 어떤 URL로 들어와도 `bingo_game.html`로 연결됩니다.

## 🔓 8단계: 데이터베이스/스토리지 보안 규칙 설정

수업 중 누구나 읽기/쓰기가 가능하도록 규칙을 풀어줍니다.

### Realtime Database 규칙
1. https://console.firebase.google.com/project/bing-go-game/database 접속
2. **규칙(Rules)** 탭 클릭
3. 아래 내용으로 교체:
   ```json
   {
     "rules": {
       ".read": true,
       ".write": true
     }
   }
   ```
4. **게시(Publish)** 클릭

### Storage 규칙
1. https://console.firebase.google.com/project/bing-go-game/storage 접속
2. **규칙(Rules)** 탭 클릭
3. 아래로 교체:
   ```
   rules_version = '2';
   service firebase.storage {
     match /b/{bucket}/o {
       match /{allPaths=**} {
         allow read, write: if true;
       }
     }
   }
   ```
4. **게시(Publish)** 클릭

> ⚠️ 이 설정은 **수업용 임시 설정**입니다. 수업이 끝나면 다시 잠그세요.

## 🚀 9단계: 배포!

명령 프롬프트에서:

```bash
firebase deploy --only hosting
```

⏰ 30초 ~ 1분 소요.

성공하면 마지막 줄에 이렇게 나옵니다:

```
✔  Deploy complete!

Hosting URL: https://bing-go-game.web.app
```

🎉 **이 URL이 학생들에게 공유할 링크**예요!

---

## 🎮 10단계: 테스트

1. 위 URL을 PC 브라우저에서 열기 → **선생님으로 입장**
2. 휴대폰 또는 다른 브라우저로 같은 URL 열기 → **학생으로 입장**
3. 선생님 화면에서:
   - 빙고 크기 선택 (예: 4x4)
   - 이미지 16장 이상 업로드 (파일명이 단어가 됨)
   - "🚀 빙고 보드 생성 & 게임 시작" 클릭
4. 이미지 그리드에서 그림 클릭 → 학생 화면에 2초간 표시
5. 학생은 자기 빙고판에서 같은 단어 클릭 → 체크
6. 가로/세로/대각선 한 줄 완성 → 🎉 BINGO!

## ❓ 문제 해결

### "이미 선생님이 입장 중입니다" 에러
→ 페이지 새로고침 시 정상. 또는 게임 화면 우측 상단 "🔄 게임 초기화" 버튼.

### 학생이 입장은 했는데 빙고판이 안 보임
→ 선생님이 "🚀 게임 시작" 버튼을 눌러야 보드가 생성됩니다. 학생이 들어오기 전에 시작했다면 다시 한 번 눌러주세요.

### 이미지 업로드 실패
→ Storage 규칙(8단계)이 제대로 게시됐는지 확인. CORS 에러면 5분 정도 기다린 후 재시도.

### 모바일에서 글자가 너무 작음
→ 5x5는 단어가 짧아야 잘 보입니다. 3x3 또는 4x4 추천.

### 같은 시간에 다른 반과 동시에 쓰고 싶다면
→ URL 뒤에 `?room=2학년3반` 같이 붙이면 방이 분리됩니다.
